# E1.1: Auth and sign-up

**Status:** active
**Assignee:** ai
**Planned:** 2026-06-25

## Goal

Email + password sign-up with sessions stored in httpOnly cookies.

## Context

This is the gate for everything else: no auth, no per-user data. Keep it boring
and standard. Tokens never touch the client; the frontend never decodes them.
Do not build social login or password reset here — those are separate epics.

## Tasks

- E1.1-T1: Users table + migration. Email unique, hashed password (argon2).
- E1.1-T2: Sign-up + login handlers. Rate-limit login. Return a session cookie.
- E1.1-T3: Session middleware that loads the user on each request.

## Exit

A new email registers, logs in, and stays logged in across a page refresh.
